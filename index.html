Chrome Extension Upload Process

Step 1: Prepare Files

  • Zip your entire source code into a .zip file
  • Ensure manifest.json is in the root folder
  • .zip file should not exceed 20MB
  • Remove unnecessary files (node_modules, .git, etc.)

Step 2: Access Chrome Developer Dashboard

  • Link: https://chrome.google.com/webstore/devconsole
  • Sign in with your Google account
  • Pay one-time $5 fee (if first time)

Step 3: Upload Extension

  • Click "New Item"
  • Upload .zip file
  • Wait for system validation (checks manifest errors, permissions, etc.)

Step 4: Complete Store Listing Information

  • Detailed description: 132-10,000 characters (recommended 500-1000 characters)
  • Short description: 132 characters max (displayed in search)
  • Category: Choose the most appropriate category
  • Language: Primary language of your extension
  • Icons & Images: Upload all prepared assets

Step 5: Privacy & Distribution

  • Visibility: Public / Unlisted / Private
  • Privacy Policy: URL required if collecting user data
  • Regions: Select countries/regions for distribution

Step 6: Submit for Review

  • Review all information
  • Click "Submit for Review"
  • Review time: few hours to several days (typically 1-3 days)
  • You'll receive email notification when approved or rejected

Best Practices

WARNING
  • Screenshots should have clear annotations about features
  • Description should include keywords for better SEO
  • Icon must be clear and recognizable at all sizes
  • Promotional images must be professional, without overly small text
  • Privacy policy must be clear about data handling

Common Rejection Reasons

  • Unreasonable permissions: Requesting unnecessary permissions
  • Unclear description: Not explaining functionality sufficiently
  • Missing privacy policy: Collecting data without policy
  • Spam/misleading: Deceptive description or keyword spam
  • Copyright: Using unauthorized logos/brand names
  • Malware/phishing: Malicious code or scam
